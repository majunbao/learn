# 05. Spring Security + JWT Authentication

## In One Sentence

Add Spring Security and JWT to the project so users can log in with username/password, get a token, and use that token to access protected APIs — with role-based permission checking against the 5 permission tables.

---

## What We're Building

```
POST /login  (username + password)
      │
      ▼
  Authenticate against hr table
      │
      ▼
  Issue JWT token containing user info + roles
      │
      ▼
  Every subsequent request: Authorization: Bearer <token>
      │
      ▼
  JWT filter extracts token → loads user → loads roles → loads menus
      │
      ▼
  Spring Security checks: does this role have access to this URL?
```

## Step 3: Write the JWT Utility Class

Create `JwtUtils.java` under `com.vhr.security`:

```java
package com.vhr.security;

import io.jsonwebtoken.Claims;
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import io.jsonwebtoken.security.Keys;

import java.security.Key;
import java.util.Date;
import java.util.HashMap;
import java.util.Map;

public class JwtUtils {

    // In production, store this in a config file or environment variable — never hardcode in real projects
    private static final Key SECRET_KEY = Keys.secretKeyFor(SignatureAlgorithm.HS256);

    // Token expires in 24 hours
    private static final long EXPIRATION_MS = 24 * 60 * 60 * 1000;

    /**
     * Generate a JWT token for a given username.
     * The token payload includes the username and issue time.
     */
    public static String generateToken(String username) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("username", username);

        return Jwts.builder()
                .setClaims(claims)
                .setSubject(username)
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + EXPIRATION_MS))
                .signWith(SECRET_KEY)
                .compact();
    }

    /**
     * Parse a JWT token and extract the claims (payload).
     * Throws an exception if the token is expired, malformed, or has an invalid signature.
     */
    public static Claims parseToken(String token) {
        return Jwts.parserBuilder()
                .setSigningKey(SECRET_KEY)
                .build()
                .parseClaimsJws(token)
                .getBody();
    }

    /**
     * Extract the username from a JWT token.
     */
    public static String getUsernameFromToken(String token) {
        return parseToken(token).getSubject();
    }

    /**
     * Check whether a token has expired.
     */
    public static boolean isTokenExpired(String token) {
        return parseToken(token).getExpiration().before(new Date());
    }

    /**
     * Validate a token: not expired and username matches.
     */
    public static boolean validateToken(String token, String username) {
        String tokenUsername = getUsernameFromToken(token);
        return tokenUsername.equals(username) && !isTokenExpired(token);
    }
}
```

### JWT Explained

| Concept | What it means |
|---------|---------------|
| **Header** | Metadata: algorithm (HS256) and token type (JWT) |
| **Payload (claims)** | The data you want to carry: username, roles, expiry time |
| **Signature** | A cryptographic hash of header + payload using the secret key — proves the token wasn't tampered with |
| **Why stateless?** | The server doesn't need to store session data. The token itself contains everything needed to identify the user. The server just verifies the signature. |

> A JWT looks like: `eyJhbGciOi... . eyJzdWIiOi... . SflKxwRJS...` — three Base64-encoded parts separated by dots. You can decode the first two parts on [jwt.io](https://jwt.io) to see the header and payload. The third part is the signature.

## Step 4: Write the Login Endpoint

Create `LoginController.java` under `com.vhr.controller`:

```java
package com.vhr.controller;

import com.vhr.bean.Hr;
import com.vhr.security.JwtUtils;
import com.vhr.service.HrService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;

import java.util.HashMap;
import java.util.Map;

@RestController
public class LoginController {

    @Autowired
    private HrService hrService;

    /**
     * POST /login
     * Accepts JSON: { "username": "admin", "password": "123" }
     * Returns JSON: { "token": "eyJ...", "user": { ... } }
     */
    @PostMapping("/login")
    public Map<String, Object> login(@RequestBody Map<String, String> loginRequest) {
        String username = loginRequest.get("username");
        String password = loginRequest.get("password");

        // 1. Authenticate: look up the user and verify the password
        Hr hr = hrService.authenticate(username, password);

        // 2. Generate a JWT token
        String token = JwtUtils.generateToken(username);

        // 3. Return the token and user info
        Map<String, Object> result = new HashMap<>();
        result.put("token", token);
        result.put("user", hr);
        return result;
    }
}
```

### What's happening here?

1. The frontend sends `POST /login` with `{ "username": "admin", "password": "123" }`.
2. `HrService.authenticate()` looks up the user in the `hr` table and checks the password.
3. If authentication succeeds, `JwtUtils.generateToken()` creates a signed JWT.
4. The frontend stores this token and sends it in the `Authorization` header for every subsequent request.

### 5.4 HrServiceImpl

Create `HrServiceImpl.java` under `com.vhr.service.impl`:

```java
package com.vhr.service.impl;

import com.vhr.bean.Hr;
import com.vhr.mapper.HrMapper;
import com.vhr.service.HrService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.stereotype.Service;

@Service
public class HrServiceImpl implements HrService {

    @Autowired
    private HrMapper hrMapper;

    private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

    @Override
    public Hr authenticate(String username, String password) {
        // 1. Look up the user by username
        Hr hr = hrMapper.loadUserByUsername(username);

        // 2. Check: user exists?
        if (hr == null) {
            throw new RuntimeException("User not found: " + username);
        }

        // 3. Check: account enabled?
        if (hr.getEnabled() == null || !hr.getEnabled()) {
            throw new RuntimeException("Account is disabled: " + username);
        }

        // 4. Check: password matches?
        //    The vhr seed data uses BCrypt-hashed passwords, so we must use
        //    BCryptPasswordEncoder.matches() instead of plaintext comparison.
        if (!passwordEncoder.matches(password, hr.getPassword())) {
            throw new RuntimeException("Invalid password");
        }

        // 5. Clear the password before returning the user object to the frontend
        hr.setPassword(null);

        return hr;
    }
}
```

> **Why BCrypt?** The vhr seed data stores passwords as BCrypt hashes
> (e.g. `$2a$10$ySG2lkvjFHY5O0./CPIE1OI8VJsuKYEzOYzqIa7AJR6sEgSzUFOAm`).
> `BCryptPasswordEncoder.matches(rawPassword, hashedPassword)` compares
> the raw input against the stored hash — it is **not** a plaintext
> `equals()` call. `BCryptPasswordEncoder` comes from
> `spring-security-crypto`, which is a transitive dependency of
> `spring-boot-starter-security` — no extra dependency needed.

## Step 6: Write the JWT Authentication Filter

This is the heart of the system. Every HTTP request passes through this filter.

Create `JwtAuthenticationFilter.java` under `com.vhr.security`:

```java
package com.vhr.security;

import com.vhr.bean.Hr;
import com.vhr.mapper.HrMapper;
import io.jsonwebtoken.Claims;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;
import org.springframework.security.core.context.SecurityContextHolder;
import org.springframework.stereotype.Component;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.FilterChain;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.ArrayList;

@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private HrMapper hrMapper;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        // 1. Extract the Authorization header
        String authHeader = request.getHeader("Authorization");

        // 2. If no token or wrong format, skip — let Spring Security handle it (likely 401)
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        // 3. Extract the token (remove "Bearer " prefix)
        String token = authHeader.substring(7);

        try {
            // 4. Parse the token
            Claims claims = JwtUtils.parseToken(token);
            String username = claims.getSubject();

            // 5. Load the user from the database
            Hr hr = hrMapper.loadUserByUsername(username);

            if (hr != null && hr.getEnabled()) {
                // 6. Create an authentication object and set it in the security context
                //    This tells Spring Security: "this request is authenticated as user X"
                UsernamePasswordAuthenticationToken authentication =
                        new UsernamePasswordAuthenticationToken(hr, null, new ArrayList<>());
                SecurityContextHolder.getContext().setAuthentication(authentication);
            }
        } catch (Exception e) {
            // Token is invalid, expired, or user not found — don't authenticate
            // Spring Security will treat this as an unauthenticated request
        }

        // 7. Continue the filter chain
        filterChain.doFilter(request, response);
    }
}
```

### Filter Flow Explained

```
Request arrives
      │
      ▼
  Has "Authorization: Bearer <token>" header?
      │
      ├── No  → skip, let Spring Security reject (401)
      │
      └── Yes → parse token → extract username
                    │
                    ▼
              Load user from hr table
                    │
                    ▼
              User exists and enabled?
                    │
                    ├── No  → skip, let Spring Security reject (401)
                    │
                    └── Yes → set authentication in SecurityContext
                                  │
                                  ▼
                            Continue to the controller
```

## Step 7: Configure Spring Security

Create `SecurityConfig.java` under `com.vhr.security`:

```java
package com.vhr.security;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter;

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Autowired
    private JwtAuthenticationFilter jwtAuthenticationFilter;

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
                // 1. Disable CSRF — we're using JWT, not cookies, so CSRF is not a threat
                .csrf().disable()

                // 2. Stateless session — no HttpSession, every request carries its own token
                .sessionManagement()
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)

                .and()

                // 3. Define access rules
                .authorizeRequests()
                // Public endpoints: anyone can access
                .antMatchers("/login", "/hello").permitAll()
                // Everything else requires authentication
                .anyRequest().authenticated()

                .and()

                // 4. Return 401 (not 403) when no token is provided
                //    Without this, Spring Security defaults to Http403ForbiddenEntryPoint
                //    which returns 403 even for anonymous users — confusing for REST APIs.
                .exceptionHandling()
                .authenticationEntryPoint((request, response, authException) -> {
                    response.setStatus(401);
                    response.setContentType("application/json");
                    response.getWriter().write("{\"status\":401,\"error\":\"Unauthorized\",\"message\":\"Please provide a valid JWT token\"}");
                })

                .and()

                // 5. Insert our JWT filter BEFORE Spring Security's default UsernamePasswordAuthenticationFilter
                //    This ensures the JWT is checked first, before any other auth mechanism
                .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    }
}
```

### Configuration Explained

| Setting | What it does | Why |
|---------|-------------|-----|
| `csrf().disable()` | Disables CSRF protection | JWT is sent in the `Authorization` header, not in cookies. CSRF attacks rely on cookies, so this protection is unnecessary and would block legitimate API calls. |
| `sessionManagement().stateless()` | No server-side session | Every request is self-contained with its JWT. The server doesn't track login state. |
| `.antMatchers("/login", "/hello").permitAll()` | Allow unauthenticated access | `/login` must be public (you can't require a token to get a token). `/hello` is our test endpoint. |
| `.anyRequest().authenticated()` | Everything else needs a token | All other endpoints (like `/menu/all`) require a valid JWT. |
| `.exceptionHandling().authenticationEntryPoint(...)` | Return 401 instead of 403 | Spring Security's default for anonymous users hitting a protected endpoint is **403 Forbidden**. For a REST API, **401 Unauthorized** is the correct response — it tells the client "you need to authenticate first". |
| `.addFilterBefore(jwtFilter, ...)` | Run our filter first | Extract the JWT and set the security context before Spring Security checks permissions. |

### 8.1 Start the project

Make sure MySQL is running, then start `VhrApplication`.

### 8.2 Test: access a protected endpoint without a token

```bash
curl http://localhost:8082/menu/all
```

Expected: `401 Unauthorized` or a Spring Security default login page redirect.

### 8.3 Test: login to get a token

```bash
curl -X POST http://localhost:8082/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123"}'
```

Expected response (token truncated for readability):

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VybmFtZSI6ImFkbWluIiwi...",
  "user": {
    "id": 1,
    "name": "System Admin",
    "username": "admin",
    "enabled": true
  }
}
```

### 8.4 Test: access a protected endpoint with the token

Copy the token from the login response and use it:

```bash
curl http://localhost:8082/menu/all \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.eyJ1c2VybmFtZSI6ImFkbWluIiwi..."
```

Expected: the full menu list from the database.

### 8.5 Test: access with an expired or invalid token

```bash
curl http://localhost:8082/menu/all \
  -H "Authorization: Bearer invalidtoken123"
```

Expected: `401 Unauthorized`.
