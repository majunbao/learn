# 02. Docker MySQL Setup and Data Import

## In One Sentence

Use Docker to run a MySQL 8.0 container, create a database called `vhr`, and import the tables and seed data from `resources/vhr.sql`. **After completing this guide, you'll have a MySQL instance on your machine with real data that Spring Boot can connect to.**

---

## 2.1 Start the MySQL Container

> This step only needs to be done once. Afterwards, just run `docker start mysql-vhr` to bring it back up.

**Mac / Linux:**

```bash
docker run -d \
  --name mysql-vhr \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123 \
  -v mysql-vhr-data:/var/lib/mysql \
  mysql:8.0 \
  --default-authentication-plugin=mysql_native_password \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

**Windows (PowerShell):**

```powershell
docker run -d `
  --name mysql-vhr `
  -p 3306:3306 `
  -e MYSQL_ROOT_PASSWORD=123 `
  -v mysql-vhr-data:/var/lib/mysql `
  mysql:8.0 `
  --default-authentication-plugin=mysql_native_password `
  --character-set-server=utf8mb4 `
  --collation-server=utf8mb4_unicode_ci
```

> PowerShell's line continuation character is the backtick **`` ` ``** (the key to the left of `1`, sharing the `~` key). The backtick **must be the last character on the line** — no trailing spaces.

**Windows (cmd):**

```cmd
docker run -d ^
  --name mysql-vhr ^
  -p 3306:3306 ^
  -e MYSQL_ROOT_PASSWORD=123 ^
  -v mysql-vhr-data:/var/lib/mysql ^
  mysql:8.0 ^
  --default-authentication-plugin=mysql_native_password ^
  --character-set-server=utf8mb4 ^
  --collation-server=utf8mb4_unicode_ci
```

### What Each Parameter Does

| Parameter | Purpose |
| :--- | :--- |
| `-d` | Run in background (detached) |
| `--name mysql-vhr` | Name the container `mysql-vhr` for easy reference |
| `-p 3306:3306` | Map the container's port 3306 to your machine's port 3306 |
| `-e MYSQL_ROOT_PASSWORD=123` | Set the root password to `123` (Spring Boot config will use this too) |
| `-v mysql-vhr-data:/var/lib/mysql` | **Critical**: store MySQL data in a Docker named volume so data survives container deletion |
| `mysql:8.0` | Use the MySQL 8.0 image (has native ARM64 and x64 builds — works on Apple Silicon and Intel) |
| `--default-authentication-plugin=mysql_native_password` | Required for MySQL 8.0. The vhr project's older JDBC driver doesn't understand 8.0's new auth plugin; this forces the old one |
| Last two charset lines | Set the database default charset to utf8mb4 (supports Chinese characters) |

## 2.2 Create the vhr Database

Wait 10–30 seconds (MySQL takes a moment on first start), then run:

```bash
docker exec -i mysql-vhr mysql -uroot -p123 -e "CREATE DATABASE IF NOT EXISTS vhr DEFAULT CHARACTER SET utf8mb4;"
```

> You may see a yellow warning: `mysql: [Warning] Using a password on the command line interface can be insecure.` — this is **normal**, not an error. MySQL is just reminding you that typing a password on the command line isn't secure. The command has already executed successfully. If you see `Query OK, 1 row affected` the database was created; if you don't see it, the database already existed (MySQL doesn't print `Query OK` when the database already exists). Verify with:
>
> ```bash
> docker exec -i mysql-vhr mysql -uroot -p123 -e "SHOW DATABASES LIKE 'vhr';"
> ```
>
> If you see a row with `vhr`, you're good.

## 2.3 Import vhr.sql to Create Tables

The `vhr` database is empty. Now import the tables and seed data.

**Make sure your terminal is in the project root** (the directory containing `resources/`), then pick the command for your OS:

**Mac / Linux:**

```bash
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources/vhr.sql
```

**Windows (PowerShell):**

```powershell
Get-Content resources\vhr.sql | docker exec -i mysql-vhr mysql -uroot -p123 vhr
```

**Windows (cmd):**

```cmd
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources\vhr.sql
```

> This may take a few dozen seconds (vhr.sql is fairly long). **No error output** means success.

---

## 2.4 Verify the Tables Were Created

Run the following to check that all tables exist:

```bash
docker exec -i mysql-vhr mysql -uroot -p123 -e "USE vhr; SHOW TABLES;"
```

You should see about 20 tables (hr, role, menu, hr_role, menu_role, department, employee, etc.).

Then confirm the core permission tables have data:

```bash
docker exec -i mysql-vhr mysql -uroot -p123 -e "USE vhr; SELECT COUNT(*) AS hr_count FROM hr; SELECT COUNT(*) AS role_count FROM role;"
```

Non-zero numbers mean the seed data was imported. **The MySQL side is done.**

## 2.5 If You Messed Up and Want to Start Over

```bash
# 1. Drop the database
docker exec -i mysql-vhr mysql -uroot -p123 -e "DROP DATABASE vhr;"

# 2. Recreate it
docker exec -i mysql-vhr mysql -uroot -p123 -e "CREATE DATABASE IF NOT EXISTS vhr DEFAULT CHARACTER SET utf8mb4;"

# 3. Re-import the SQL (Mac/Linux version; use the platform-specific command above for Windows)
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources/vhr.sql
```

## 2.6 Common Command Reference

| Action | Command |
| :--- | :--- |
| Check if MySQL container is running | `docker ps` (seeing `mysql-vhr` means it's running) |
| View startup logs (for troubleshooting) | `docker logs mysql-vhr` |
| Stop MySQL | `docker stop mysql-vhr` |
| Start MySQL (after a reboot) | `docker start mysql-vhr` |
| Restart MySQL | `docker restart mysql-vhr` |
| Delete container (data survives) | `docker rm -f mysql-vhr` |
| Completely delete the data volume (⚠️ destructive) | `docker volume rm mysql-vhr-data` |
| Delete the MySQL image (must delete container first) | `docker rmi mysql:8.0` |
