# 03. Spring Boot Project Setup

## In One Sentence

Create a Spring Boot project in IntelliJ IDEA using Maven, write a simple `/hello` endpoint, start the project, and access it in the browser. **After completing this guide, you'll have a working Spring Boot skeleton that can connect to a database and serve APIs** — ready for adding business features on top.

> Prerequisite: the MySQL container from guide `02` is running and the database has tables.

---

## 3.1 Prerequisite Check: Is MySQL Running?

First, confirm MySQL is alive:

```bash
docker ps
```

You should see `mysql-vhr` in the list. If not, start it:

```bash
docker start mysql-vhr
```

---

## 3.2 Create a Maven Project in IntelliJ IDEA

IntelliJ IDEA Community Edition doesn't have `Spring Initializr`, so we'll create a plain Maven project and add Spring Boot dependencies manually.

1. Open IntelliJ IDEA → `File` → `New` → `Project...`
2. Fill in the project info:
   - **Name**: `vhr-learn`
   - **Location**: pick a directory you can find, e.g. `D:\codes\vhr-learn`
   - **Language**: `Java`
   - **Build system**: `Maven`
   - **JDK**: choose **JDK 11** (Spring Boot 2.1.3 supports JDK 8 or 11; 11 is recommended)
   - **Add sample code**: **uncheck** (we'll write our own code from scratch)
3. Click `Create` and wait for IntelliJ IDEA to initialize

---

## 3.3 Set Up the Directory Structure

### 3.3.1 Create Java packages

Under `src/main/java`, **manually create the package** `com.vhr` (right-click `java` → `New` → `Package` → type `com.vhr`).

### 3.3.2 Create the controller directory

Under `com.vhr`, create the `controller` directory (right-click `com.vhr` → `New` → `Package` → type `controller`).

### 3.3.3 Create the resources directory

If `src/main` doesn't have a `resources` directory, create one (right-click `src/main` → `New` → `Directory` → type `resources`), then **right-click it → Mark Directory as → Resources Root**.

Your project structure should now look like:

```
vhr-learn/
├── src/
│   └── main/
│       ├── java/
│       │   └── com/
│       │       └── vhr/
│       │           └── controller/
│       └── resources/
└── pom.xml
```

---

## 3.4 Modify pom.xml

Open `pom.xml` and replace the entire file with:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <!-- Inherit Spring Boot's parent POM: manages all dependency versions, avoids version conflicts -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.3.RELEASE</version>
        <relativePath/>
    </parent>

    <!-- Project metadata -->
    <groupId>com.vhr</groupId>
    <artifactId>vhr-learn</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <name>vhr-learn</name>
    <description>vhr learning project</description>

    <!-- Properties -->
    <properties>
        <java.version>11</java.version>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <!-- Dependencies -->
    <dependencies>
        <!-- Spring Boot Web: provides embedded Tomcat + Spring MVC for writing APIs -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- MySQL driver: lets Java connect to MySQL -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <scope>runtime</scope>
        </dependency>

        <!-- Druid: Alibaba's database connection pool for better DB access performance -->
        <dependency>
            <groupId>com.alibaba</groupId>
            <artifactId>druid-spring-boot-starter</artifactId>
            <version>1.1.16</version>
        </dependency>

        <!-- MyBatis Plus: enhanced MyBatis toolkit for simpler database operations -->
        <dependency>
            <groupId>com.baomidou</groupId>
            <artifactId>mybatis-plus-boot-starter</artifactId>
            <version>3.1.0</version>
        </dependency>
    </dependencies>

    <!-- Build configuration -->
    <build>
        <plugins>
            <!-- Spring Boot Maven Plugin: provides packaging and running capabilities -->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
```

After editing, IntelliJ IDEA will show a Maven refresh button in the top-right corner. Click it to download dependencies (the first download may take a few minutes). Once you see `BUILD SUCCESS` in the bottom-right, you can continue.

## 3.5 Write the Application Class VhrApplication

Create a new Java class in the `com.vhr` package:

```java
package com.vhr;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class VhrApplication {

    public static void main(String[] args) {
        SpringApplication.run(VhrApplication.class, args);
        System.out.println("(♥◠‿◠)ﾉﾞ  vhr started successfully   ლ(´ڡ`ლ)ﾞ");
    }
}
```

File path: `src/main/java/com/vhr/VhrApplication.java`

---

## 3.6 Write the Configuration File application.properties

Create `application.properties` in the `resources` directory:

```properties
# ================================
# Server Port
# ================================
server.port=8082

# ================================
# Database Connection
# ================================
spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
spring.datasource.username=root
spring.datasource.password=123
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/vhr?characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

File path: `src/main/resources/application.properties`

## 3.7 Write the Test Endpoint HelloController

Create `HelloController` in the `com.vhr.controller` package:

```java
package com.vhr.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello vhr!";
    }
}
```

---

## 3.8 Start the Project and Verify in the Browser

1. **First, confirm MySQL is still running**

   ```bash
   docker ps
   ```

2. **Start Spring Boot in IntelliJ IDEA**

   Make sure the currently open file is `VhrApplication.java`, then click the green ▶️ run button in the toolbar — or use the shortcut Shift+F10.

3. **Check the console for key messages**

   On successful startup you'll see:
   - `Started VhrApplication in X.XXX seconds` (startup complete)
   - `Tomcat started on port(s): 8082 (http)` (Tomcat listening on 8082)
   - The last line: `(♥◠‿◠)ﾉﾞ  vhr started successfully   ლ(´ڡ`ლ)ﾞ`

4. **Verify in the browser**

   Open your browser and visit: `http://localhost:8082/hello`

   Seeing `Hello vhr!` means the entire pipeline works. **Spring Boot starts normally + MySQL connects successfully**.

## 3.9 Troubleshooting

| Symptom | Likely Cause | Diagnostic Command / Action |
| :--- | :--- | :--- |
| Startup error: `Failed to configure a DataSource` | Can't connect to the database | ① `docker ps` to confirm mysql-vhr is running; ② check url/password in `application.properties` |
| Startup error: `Communications link failure` | MySQL container not running or wrong port | `docker ps` to confirm container is running; `docker logs mysql-vhr` to check MySQL startup logs |
| Browser can't reach `localhost:8082/hello` | ① Project didn't start ② Port is taken by another process | Check IntelliJ IDEA console for errors; `netstat -ano | findstr 8082` to see who's using the port |
| API responds but data is wrong | We haven't connected to the database tables yet (this guide only verifies "it runs") | The next guide starts writing real business code, connecting tables via mappers |
