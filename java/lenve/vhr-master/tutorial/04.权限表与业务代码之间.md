# 04. 权限表与业务代码之间

## 一句话说清这篇要干什么

把 vhr 数据库里的 5 张权限核心表和 Java 代码串起来，写一个简单的查询接口，看看数据是怎么流动的。

## 核心权限表说明

vhr 的权限系统由 5 张表组成：

| 表名          | 说明        |
| ----------- | --------- |
| `hr`        | 操作员表（用户表） |
| `role`      | 角色表       |
| `menu`      | 菜单/权限表    |
| `hr_role`   | 操作员-角色关联表 |
| `menu_role` | 角色-菜单关联表  |

关系图：

```
hr ←→ hr_role ←→ role ←→ menu_role ←→ menu
```

## 第一步：创建目录结构

先在项目里建好下面的目录（注意创建顺序）：

1. 先在 `com.vhr` 下创建：`bean`、`mapper`、`service`、`controller` 这四个同级目录
2. 再在 `service` 目录里面创建 `impl` 子目录

最终结构：

```
src/main/java/com/vhr
├── bean          # 目录，实体类
├── mapper        # 目录，Mapper 接口
├── service       # 目录，Service 层接口
│   └── impl      # service 下面的子目录，Service 层实现
└── controller    # 目录，Controller 层
```

### 各层对比说明

| Java Spring Boot  | 对应 Node.js (Express/Koa)        | 对应 Vue                | 作用                   |
| ----------------- | ------------------------------- | --------------------- | -------------------- |
| `bean` / `entity` | `model` / `entity`              | `data` / 组件 `props`   | 定义数据结构，和数据库表对应       |
| `mapper`          | `dao` / `repository`            | -                     | 写 SQL，和数据库交互         |
| `service`         | `service`                       | `store` / composables | 写业务逻辑，处理数据           |
| `controller`      | `controller` / `router handler` | -                     | 接收请求，调用 service，返回响应 |

## 第二步：写实体类（Bean）

> 注意：这里的 "bean" 是目录名，叫"实体类"的意思；而"Spring Bean"（大写 B）是 Spring 管理的对象，两个概念相关但不一样！我们现在写的实体类暂时还没给 Spring 管理，所以不需要加 `@Component` 注解。

## 第三步：写 Mapper 接口和 XML

### 1. MenuMapper 接口

在 `com.vhr.mapper` 下新建 `MenuMapper.java`：

```java
package com.vhr.mapper;

import com.vhr.bean.Menu;
import org.apache.ibatis.annotations.Mapper;

import java.util.List;

@Mapper
public interface MenuMapper {
    List<Menu> getAllMenus();
}
```

### 2. MenuMapper.xml

先在 `src/main/resources` 下新建 `mapper` 目录，然后在里面新建 `MenuMapper.xml`：

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.vhr.mapper.MenuMapper">

    <resultMap id="BaseResultMap" type="com.vhr.bean.Menu">
        <id property="id" column="id"/>
        <result property="url" column="url"/>
        <result property="path" column="path"/>
        <result property="component" column="component"/>
        <result property="name" column="name"/>
        <result property="iconCls" column="iconCls"/>
        <result property="keepAlive" column="keepAlive"/>
        <result property="requireAuth" column="requireAuth"/>
        <result property="parentId" column="parentId"/>
        <result property="enabled" column="enabled"/>
    </resultMap>

    <select id="getAllMenus" resultMap="BaseResultMap">
        SELECT * FROM menu WHERE enabled = 1
    </select>

</mapper>
```

#### Mapper XML 详细解释

| 部分                                      | 说明                                                       | 对应 Node.js                               |
| --------------------------------------- | -------------------------------------------------------- | ---------------------------------------- |
| `namespace="com.vhr.mapper.MenuMapper"` | 关联到 Java 接口 `MenuMapper`，MyBatis 会根据这个找到要实现的接口           | -                                        |
| `<resultMap>`                           | 定义数据库表列名和 Java 实体类属性名的映射关系                               | 类似于 ORM 库（如 Sequelize/TypeORM）的 model 定义 |
| `<select id="getAllMenus">`             | `id` 必须和 Java 接口里的方法名一致，MyBatis 会自动把 SQL 查询结果转换成 Java 对象 | 对应 Node.js 里的 `dao.getAllMenus()` 方法     |

#### 简单对比 Node.js

**Node.js 写法（比如用 mysql2）：**

```javascript
// dao/menuDao.js
async function getAllMenus() {
  const [rows] = await connection.execute('SELECT * FROM menu WHERE enabled = 1');
  // 手动把数据库结果转换成对象
  return rows.map(row => ({
    id: row.id,
    url: row.url,
    // ... 手动映射每个字段
  }));
}
```

**MyBatis 写法（上面的 XML）：**

```xml
<select id="getAllMenus" resultMap="BaseResultMap">
    SELECT * FROM menu WHERE enabled = 1
</select>
```

区别：MyBatis 通过 `<resultMap>` 自动帮你把数据库的列映射成 Java 对象，不需要手动一行行写。

### 3. 修改 application.properties，添加 MyBatis-Plus 配置

打开 `src/main/resources/application.properties`，在最后添加：

```properties
# ================================
# MyBatis-Plus
# ================================
# Location of MyBatis Mapper XML files
mybatis-plus.mapper-locations=classpath:mapper/*.xml
# Print executed SQL in console for debugging
mybatis-plus.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl
```

> **What do these do?**
> 1. `mybatis-plus.mapper-locations`: Tells MyBatis where to find our `MenuMapper.xml` file
> 2. `mybatis-plus.configuration.log-impl`: Prints the executed SQL statements in the IDEA console, which makes debugging easier

## 第四步：写 Service 层

### 1. MenuService 接口

在 `com.vhr.service` 下新建 `MenuService.java`：

```java
package com.vhr.service;

import com.vhr.bean.Menu;

import java.util.List;

public interface MenuService {
    List<Menu> getAllMenus();
}
```

### 2. MenuServiceImpl 实现类

在 `com.vhr.service.impl` 下新建 `MenuServiceImpl.java`：

```java
package com.vhr.service.impl;

import com.vhr.bean.Menu;
import com.vhr.mapper.MenuMapper;
import com.vhr.service.MenuService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class MenuServiceImpl implements MenuService {

    @Autowired
    private MenuMapper menuMapper;

    @Override
    public List<Menu> getAllMenus() {
        return menuMapper.getAllMenus();
    }
}
```

#### Service 层详细解释

##### 什么是 Bean？

Bean 简单来说就是**由 Spring 管理的对象**。

**对比 Node.js：**

```javascript
// Node.js 里我们这样用对象
const menuService = require('./menuService'); // 每次 require 都要手动导入
menuService.getAllMenus();
```

**Spring 里这样用：**

```java
@Autowired
private MenuService menuService; // 不需要自己 new，Spring 会帮我们创建好
menuService.getAllMenus();
```

**对应 Vue：**
Bean 有点像 Vue 的全局注册组件，或者 Pinia store —— 你在一个地方注册，在任何地方都能直接用，不需要每次都重新创建。

| Java Spring Bean                          | 对应 Node.js                | 对应 Vue                   |
| ----------------------------------------- | ------------------------- | ------------------------ |
| `@Service` / `@Component` / `@Controller` | `module.exports` 导出模块     | Vue 插件全局注册 / Pinia store |
| `@Autowired` 注入                           | `require()` / `import` 导入 | `useStore()` 或者 `inject` |

| 部分           | 说明                                                              | 对应 Node.js                                |
| ------------ | --------------------------------------------------------------- | ----------------------------------------- |
| `@Service`   | 告诉 Spring "把这个类交给你管理，注册成一个 Bean"，后面我们可以在任何地方通过 `@Autowired` 获取它 | -                                         |
| `@Autowired` | 自动注入依赖，Spring 会帮我们创建 `MenuMapper` 的实例并赋值给 `menuMapper`          | 对应 Node.js 里的 `require('./menuDao')` 导入模块 |
| `@Override`  | 表示这个方法是实现了接口 `MenuService` 里的方法                                 | -                                         |

#### Service 层对比 Node.js

**Node.js 写法：**

```javascript
// service/menuService.js
const menuDao = require('../dao/menuDao'); // 对应 @Autowired

async function getAllMenus() {
  // 这里可以写业务逻辑，比如数据过滤、转换等
  const menus = await menuDao.getAllMenus();
  return menus;
}

module.exports = { getAllMenus };
```

**Java 写法（上面的代码）：**

```java
@Service
public class MenuServiceImpl implements MenuService {
    @Autowired
    private MenuMapper menuMapper; // 对应 Node.js 的 require 导入

    @Override
    public List<Menu> getAllMenus() {
        return menuMapper.getAllMenus();
    }
}
```

## 第五步：写 Controller 层

在 `com.vhr.controller` 下新建 `MenuController.java`：

```java
package com.vhr.controller;

import com.vhr.bean.Menu;
import com.vhr.service.MenuService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

@RestController
public class MenuController {

    @Autowired
    private MenuService menuService;

    @GetMapping("/menu/all")
    public List<Menu> getAllMenus() {
        return menuService.getAllMenus();
    }
}
```

#### Controller 层详细解释

| 部分                         | 说明                                                                                  | 对应 Node.js / Vue                                  |
| -------------------------- | ----------------------------------------------------------------------------------- | ------------------------------------------------- |
| `@RestController`          | 告诉 Spring 这是一个 Controller，并且所有方法的返回值会被自动转换成 JSON（相当于 `@Controller + @ResponseBody`） | 对应 Express/Koa 里的路由 handler                       |
| `@GetMapping("/menu/all")` | 定义 GET 请求的路径，浏览器访问 `/menu/all` 时会调用这个方法                                             | 对应 Express/Koa 里的 `app.get('/menu/all', handler)` |
| `@Autowired`               | 自动注入 `MenuService`                                                                  | 对应 Node.js 里的 `require('./menuService')`          |

#### Controller 层对比 Node.js

**Node.js Express 写法：**

```javascript
// controller/menuController.js
const menuService = require('../service/menuService');
const express = require('express');
const router = express.Router();

router.get('/menu/all', async (req, res) => {
    const menus = await menuService.getAllMenus();
    res.json(menus); // 返回 JSON 数据
});

module.exports = router;
```

**Java Spring Boot 写法（上面的代码）：**

```java
@RestController
public class MenuController {
    @Autowired
    private MenuService menuService; // 对应 Node.js 的 require 导入

    @GetMapping("/menu/all") // 对应 Express 的 router.get
    public List<Menu> getAllMenus() {
        return menuService.getAllMenus();
        // Spring 会自动把 List<Menu> 转换成 JSON 返回，不需要手动 res.json
    }
}
```

```

                                      ↓
浏览器 ← JSON 数据 ← MenuController ← MenuService ← MenuMapper
```
