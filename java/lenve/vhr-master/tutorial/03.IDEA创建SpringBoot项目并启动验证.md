# 03. IDEA 创建 Spring Boot 项目并启动验证

## 一句话说清这篇要干什么

在 IDEA 里用 Maven 手动创建一个 Spring Boot 项目，写上一个最简单的接口（`/hello`），启动项目，在浏览器里访问它。**做完这篇，你就有一个"能跑、能连数据库、能提供接口"的 Spring Boot 项目骨架**，后面往上面加业务功能就行。

> 💡 前提：`02` 号文档的 MySQL 已经跑起来了，数据库里已经建了表。

---

## 3.1 前提检查：MySQL 在不在跑

先执行一条命令，确认 MySQL 还活着：

```bash
docker ps
```

能看到 `mysql-vhr` 这一行就行了。如果没看到，先跑一下：

```bash
docker start mysql-vhr
```

---

## 3.2 用 IDEA 创建 Maven 项目

IDEA 社区版没有 `Spring Initializr`，我们用 **Maven Archetype** 来创建一个最朴素的 Java 项目，然后手动加 Spring Boot 依赖。

1. 打开 IDEA → 选 `File` → `New` → `Project...`
2. 左边选 **Maven**
3. 勾选 `Create from archetype`
4. 选择 `org.apache.maven.archetypes:maven-archetype-quickstart`
5. 点 `Next`
6. 填写项目信息：
   - **Name**：`vhr-learn`（体现"vhr 学习版"的意思）
   - **GroupId**：`com.vhr`
   - **ArtifactId**：`vhr-learn`
   - **Version**：`1.0-SNAPSHOT`（保持默认也行）
7. 点 `Next`
8. 确认 Maven 配置和 JDK（要求 JDK 8 或 JDK 11）
9. 点 `Finish`，等 IDEA 帮你生成项目结构

生成好的项目结构长这样（关键文件我标出来了）：

```
vhr-learn/
├── src/
│   └── main/
│       └── java/
│           └── com/
│               └── vhr/
│                   └── App.java        ← 等下要删掉，换成 VhrApplication
├── src/
│   └── test/                          ← 暂时用不上
└── pom.xml                            ← 等下要改
```

## 3.3 修改 pom.xml，让它变成 Spring Boot

打开 `pom.xml`，整个文件替换成下面的内容。核心变化只有两个：① 继承 Spring Boot 的父 pom，② 引入两个 starter 依赖（web 和 MySQL）。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <!-- ============================== -->
    <!-- ① 继承 Spring Boot 父 pom      -->
    <!-- 这个父 pom 里帮你定好了所有 Spring Boot -->
    <!-- 依赖的版本，所以下面 web、mysql 依赖      -->
    <!-- 都不需要写 <version>，由父 pom 统一管理     -->
    <!-- ============================== -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.3.RELEASE</version>
        <relativePath/>
    </parent>

    <groupId>com.vhr</groupId>
    <artifactId>vhr-learn</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <name>vhr-learn</name>
    <description>vhr 学习项目</description>

    <properties>
        <java.version>1.8</java.version>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <!-- ② Spring Boot Web：提供 Tomcat + Spring MVC，让你能写接口 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- ② MySQL 驱动：让 Java 程序能连 MySQL 数据库 -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <scope>runtime</scope>
        </dependency>

        <!-- ② Druid：阿里巴巴的数据库连接池（可选，但 vhr 项目用它来管理连接） -->
        <dependency>
            <groupId>com.alibaba</groupId>
            <artifactId>druid-spring-boot-starter</artifactId>
            <version>1.1.16</version>
        </dependency>

        <!-- MyBatis Plus：更简单地操作数据库（写 Mapper + 注解就能跑 SQL） -->
        <dependency>
            <groupId>com.baomidou</groupId>
            <artifactId>mybatis-plus-boot-starter</artifactId>
            <version>3.1.0</version>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
```

改完后 IDEA 右上角会出现 `Maven` 的刷新按钮，点一下让它下载依赖（第一次下载需要等几分钟）。右下角看到 `BUILD SUCCESS` 就可以继续了。

## 3.4 写启动类 VhrApplication

Spring Boot 项目**需要一个入口类**。原来 IDEA 给你生成的 `App.java` 可以删掉，新建一个文件：

```java
package com.vhr;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

// @SpringBootApplication 是个复合注解，相当于同时做三件事：
// 1. @Configuration：这个类可以放配置
// 2. @EnableAutoConfiguration：让 Spring Boot 根据依赖自动帮你配好东西
// 3. @ComponentScan：扫描 com.vhr 包下的所有 @Component / @Service / @Controller 等
@SpringBootApplication
public class VhrApplication {

    public static void main(String[] args) {
        // SpringApplication.run() 做两件事：
        // ① 创建 Spring 容器，把所有 bean 初始化好
        // ② 启动一个嵌入式的 Tomcat，监听 8082 端口（端口号下面会配）
        SpringApplication.run(VhrApplication.class, args);
        System.out.println("(♥◠‿◠)ﾉﾞ  vhr 启动成功   ლ(´ڡ`ლ)ﾞ");
    }
}
```

文件路径：`src/main/java/com/vhr/VhrApplication.java`

## 3.5 写配置文件 application.properties

Spring Boot 用这个文件配置数据库连接、端口等。它默认读取 `src/main/resources/application.properties`。

如果 `resources` 目录不存在，你需要先在 IDEA 里 `src/main/` 下新建一个 `resources` 目录，然后**右键它 → Mark Directory as → Resources Root**。

然后创建 `application.properties`：

```properties
# ================================
# 服务器端口
# ================================
server.port=8082

# ================================
# 数据库连接
# 这里的参数就是 02 号文档里启动 MySQL 时用的
# ================================
spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
spring.datasource.username=root
spring.datasource.password=123
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/vhr?characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# ================================
# MyBatis-Plus
# ================================
# 扫描 mapper 目录下的所有接口
mybatis-plus.mapper-locations=classpath*:com/vhr/mapper/*.xml
# 让控制台打印执行的 SQL（调试阶段非常有用，能看到每条语句）
mybatis-plus.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl
```

文件路径：`src/main/resources/application.properties`

### 重点字段说明

| 字段 | 是什么 |
| :--- | :--- |
| `server.port` | Spring Boot 启动后监听的端口，浏览器访问时也要用这个端口 |
| `spring.datasource.url` | 连接地址。`127.0.0.1:3306` 是本机 MySQL，`vhr` 是数据库名，后面一串参数告诉 MySQL 用 utf-8、用中国时区 |
| `spring.datasource.password` | 这个密码必须和 02 号文档里 `MYSQL_ROOT_PASSWORD=123` 保持一致 |

## 3.6 写测试接口 HelloController

在 `src/main/java/com/vhr/controller/` 下新建 `HelloController.java`（先创建 `controller` 文件夹）：

```java
package com.vhr.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

// @RestController = @Controller + @ResponseBody
// 意思是：这个类里的方法返回值会被直接作为 HTTP 响应体返回（返回 JSON 或字符串）
@RestController
public class HelloController {

    // @GetMapping 表示只处理 GET 请求
    // 浏览器访问 http://localhost:8082/hello 时就会调用这个方法
    @GetMapping("/hello")
    public String hello() {
        return "Hello vhr!";
    }
}
```

### 这几行代码干了什么

- **`@RestController`**：告诉 Spring "这个类里的方法要处理 HTTP 请求"
- **`@GetMapping("/hello")`**：把 URL 路径 `/hello` 和 `hello()` 方法绑在一起
- **`hello()` 方法的返回值**：就是 HTTP 响应体，浏览器里会直接显示 `Hello vhr!`

## 3.7 启动项目并在浏览器验证

1. **先确认 MySQL 还在跑**（再检查一次省得后面排错）

   ```bash
   docker ps | grep mysql-vhr
   ```

2. **在 IDEA 里启动 Spring Boot**

   打开 `VhrApplication.java`，点方法名左边的绿色 ▶️ 按钮，选 `Run 'VhrApplication.main()'`。

3. **看控制台的关键提示**

   启动成功你会看到：
   - `Started VhrApplication in X.XXX seconds`（启动完成）
   - `Tomcat started on port(s): 8082 (http)`（Tomcat 监听 8082）
   - 最后一行还有我们写的 `(♥◠‿◠)ﾉﾞ  vhr 启动成功`

4. **浏览器验证**

   打开浏览器访问：`http://localhost:8082/hello`

   能看到 `Hello vhr!` 就说明整个流程跑通了。**Spring Boot 能正常启动 + MySQL 能正常连上**。

## 3.8 遇到问题怎么查

| 现象 | 可能原因 | 排查命令 / 动作 |
| :--- | :--- | :--- |
| 启动时报 `Failed to configure a DataSource` | 数据库连不上 | ① `docker ps` 确认 mysql-vhr 在跑；② 检查 `application.properties` 里的 url/password |
| 启动时报 `Communications link failure` | MySQL 容器没起来或端口不对 | `docker ps` 确认容器运行，`docker logs mysql-vhr` 看 MySQL 启动日志 |
| 浏览器访问 `localhost:8082/hello` 没反应 | ① 项目没起来 ② 端口被别的程序占了 | 看 IDEA 控制台有没有报错；`lsof -i:8082`（Mac）或 `netstat -ano | findstr 8082`（Windows）看端口被谁占了 |
| 访问接口能连上但数据不对 | 还没开始连数据库表（这篇只验证"能跑"） | 下一篇开始写真正的业务代码，从 mapper 开始连表 |
