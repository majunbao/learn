# 03. IDEA 创建 Spring Boot 项目并启动验证

## 一句话说清这篇要干什么

在 IDEA 里用 Maven 手动创建一个 Spring Boot 项目，写上一个最简单的接口（`/hello`），启动项目，在浏览器里访问它。**做完这篇，你就有一个"能跑、能连数据库、能提供接口"的 Spring Boot 项目骨架**，后面往上面加业务功能就行。

> 💡 前提：`02` 号文档的 MySQL 已经跑起来了，数据库里已经建了表。

---

## 3.1 前提检查：MySQL 在不在跑

先执行一条命令，确认 MySQL 还活着：

```bash
docker ps
```

能看到 `mysql-vhr` 这一行就行了。如果没看到，先跑一下：

```bash
docker start mysql-vhr
```

---

## 3.2 用 IDEA 创建 Maven 项目

IDEA 社区版没有 `Spring Initializr`，我们直接建一个最朴素的 Maven 项目，然后手动加 Spring Boot 依赖。

1. 打开 IDEA → 选 `File` → `New` → `Project...`
2. 填写项目信息：
   - **Name**：`vhr-learn`
   - **Location**：选一个你找得到的目录，比如 `D:\codes\vhr-learn`
   - **Language**：`Java`
   - **Build system**：`Maven`
   - **JDK**：选 **JDK 11**（Spring Boot 2.1.3 支持 JDK 8 或 11，推荐用 11）
   - **Add sample code**：**不要勾选**（保持空项目，我们自己写代码）
3. 点 `Create`，等 IDEA 初始化完

---

## 3.3 建好目录结构

### 3.3.1 创建 Java 包

在 `src/main/java` 下**手动建包** `com.vhr`（右键 `java` → `New` → `Package` → 输入 `com.vhr`）。

### 3.3.2 创建 controller 目录

在 `com.vhr` 下建 `controller` 目录（右键 `com.vhr` → `New` → `Package` → 输入 `controller`）。

### 3.3.3 创建 resources 目录

如果 `src/main` 下没有 `resources` 目录，先新建一个 `resources` 目录（右键 `src/main` → `New` → `Directory` → 输入 `resources`），然后**右键它 → Mark Directory as → Resources Root**。

现在的项目结构：

```
vhr-learn/
├── src/
│   └── main/
│       ├── java/
│       │   └── com/
│       │       └── vhr/
│       │           └── controller/
│       └── resources/
└── pom.xml
```

---

## 3.4 修改 pom.xml

打开 `pom.xml`，整个文件替换成下面的内容：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <!-- 继承 Spring Boot 的父 pom：统一管理所有依赖的版本，避免版本冲突 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.3.RELEASE</version>
        <relativePath/>
    </parent>

    <!-- 项目基本信息 -->
    <groupId>com.vhr</groupId>
    <artifactId>vhr-learn</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <name>vhr-learn</name>
    <description>vhr 学习项目</description>

    <!-- 属性配置 -->
    <properties>
        <java.version>11</java.version> <!-- 项目使用 Java 11 -->
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding> <!-- 源代码编码 UTF-8 -->
    </properties>

    <!-- 项目依赖 -->
    <dependencies>
        <!-- Spring Boot Web：提供 Tomcat 服务器 + Spring MVC，让我们能写接口 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- MySQL 驱动：让 Java 程序能连接 MySQL 数据库 -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <scope>runtime</scope> <!-- 只在运行时需要，编译时不需要 -->
        </dependency>

        <!-- Druid：阿里巴巴的数据库连接池，提高数据库访问性能 -->
        <dependency>
            <groupId>com.alibaba</groupId>
            <artifactId>druid-spring-boot-starter</artifactId>
            <version>1.1.16</version>
        </dependency>

        <!-- MyBatis Plus：MyBatis 的增强工具，让操作数据库更简单 -->
        <dependency>
            <groupId>com.baomidou</groupId>
            <artifactId>mybatis-plus-boot-starter</artifactId>
            <version>3.1.0</version>
        </dependency>
    </dependencies>

    <!-- 构建配置 -->
    <build>
        <plugins>
            <!-- Spring Boot Maven 插件：提供打包、运行等功能 -->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
```

改完后 IDEA 右上角会出现 `Maven` 的刷新按钮，点一下让它下载依赖（第一次下载需要等几分钟）。右下角看到 `BUILD SUCCESS` 就可以继续了。

## 3.5 写启动类 VhrApplication

在 `com.vhr` 包下新建 Java 类：

```java
package com.vhr;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class VhrApplication {

    public static void main(String[] args) {
        SpringApplication.run(VhrApplication.class, args);
        System.out.println("(♥◠‿◠)ﾉﾞ  vhr 启动成功   ლ(´ڡ`ლ)ﾞ");
    }
}
```

文件路径：`src/main/java/com/vhr/VhrApplication.java`

---

## 3.6 写配置文件 application.properties

在 `resources` 目录下新建 `application.properties`：

```properties
# ================================
# Server Port
# ================================
server.port=8082

# ================================
# Database Connection
# ================================
spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
spring.datasource.username=root
spring.datasource.password=123
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/vhr?characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

文件路径：`src/main/resources/application.properties`

---

## 3.7 写测试接口 HelloController

在 `com.vhr.controller` 包下新建 `HelloController` 类：

```java
package com.vhr.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello vhr!";
    }
}
```

---

## 3.8 启动项目并在浏览器验证

1. **先确认 MySQL 还在跑**

   ```bash
   docker ps
   ```

2. **在 IDEA 里启动 Spring Boot**

   确认当前打开的文件是 `VhrApplication.java`，然后点 IDEA 标题栏上面的绿色 ▶️ 运行按钮 → 或者用快捷键 Shift+F10

3. **看控制台的关键提示**

   启动成功你会看到：
   - `Started VhrApplication in X.XXX seconds`（启动完成）
   - `Tomcat started on port(s): 8082 (http)`（Tomcat 监听 8082）
   - 最后一行还有我们写的 `(♥◠‿◠)ﾉﾞ  vhr 启动成功   ლ(´ڡ`ლ)ﾞ`

4. **浏览器验证**

   打开浏览器访问：`http://localhost:8082/hello`

   能看到 `Hello vhr!` 就说明整个流程跑通了。**Spring Boot 能正常启动 + MySQL 能正常连上**。

---

## 3.9 遇到问题怎么查

| 现象 | 可能原因 | 排查命令 / 动作 |
| :--- | :--- | :--- |
| 启动时报 `Failed to configure a DataSource` | 数据库连不上 | ① `docker ps` 确认 mysql-vhr 在跑；② 检查 `application.properties` 里的 url/password |
| 启动时报 `Communications link failure` | MySQL 容器没起来或端口不对 | `docker ps` 确认容器运行，`docker logs mysql-vhr` 看 MySQL 启动日志 |
| 浏览器访问 `localhost:8082/hello` 没反应 | ① 项目没起来 ② 端口被别的程序占了 | 看 IDEA 控制台有没有报错；`netstat -ano | findstr 8082` 看端口被谁占了 |
| 访问接口能连上但数据不对 | 还没开始连数据库表（这篇只验证"能跑"） | 下一篇开始写真正的业务代码，从 mapper 开始连表 |
