### 40.1 什么是 FastDFS



#### 40.1.1 FastDFS 简介



FastDFS 由淘宝的余庆大佬在 2008 年开源的一款轻量级分布式文件管理系统,FastDFS 用 C 语言实现,支持 Linux、FreeBSD、MacOS 等类 UNIX 系统。FastDFS 类似 google FS,属于应用级文件系统,不是通用的文件系统,只能通过专有 API 访问,目前提供了 C 和 Java SDK ,以及 PHP 扩展 SDK。



这款开源软件从发布至今,历经数十年,这款开源软件的生命力依然旺盛,在业界依然备受推崇,当然这也得益于作者一直在不断完善该软件。



FastDFS 专为互联网应用量身定做,解决大容量文件存储问题,追求高性能和高扩展性,它可以看做是基于文件的 key/value 存储系统,key 为文件 ID,value 为文件内容,因此称作分布式文件存储服务更为合适。



#### 40.1.2 为什么需要 FastDFS



传统的企业级开发对于高并发要求不是很高,而且数据量也不是很大,因此文件上传这种需求实现起来相对简单,直接将上传的文件保存在服务器某个目录下即可。



但是互联网应用则不同,互联网应用面向大众,数据量巨大,高并发是常态,如果还是按照传统的文件上传方案,可能存在诸多问题,例如：



1. 单机存储容量有限,无法存储海量文件



2. 单机并发能力有限,无法应对高并发上传需求



3. 单机可靠性有限,一旦服务器宕机,文件就丢失了



因此,在互联网应用中,我们需要一个分布式文件管理系统来解决这些问题。



### 40.2 FastDFS 架构



FastDFS 架构包括 Tracker Server 和 Storage Server。



#### 40.2.1 Tracker Server



Tracker Server 主要做调度工作,起到均衡的作用；负责管理所有的 Storage Server和 group,每个 Storage 在启动后会连接 Tracker,告知自己所属 group 等信息,并保持周期性心跳。



#### 40.2.2 Storage Server



Storage Server 主要提供存储服务,每个 Storage Server 会连接多个 Tracker Server,并向其报告自己的状态。



### 40.3 FastDFS 安装



FastDFS 安装比较复杂,需要安装 libfastcommon、FastDFS、FastDFS-Nginx-module、Nginx 等多个组件。具体安装步骤可以参考松哥的博客文章。
