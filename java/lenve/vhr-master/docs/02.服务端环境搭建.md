## 2.1 创建 SpringBoot 项目

> **注意**：start.spring.io 已不再支持 Spring Boot 2.x，IDEA 社区版也没有内置 Spring Initializr。
> 这里用最直接的方式：创建 Maven 项目 + 手写 `pom.xml`。

### 2.1.1 在 IDEA 社区版创建 Maven 项目

1. File → New → Project → 选择 **Maven** → Next
2. 填写：
   - **GroupId**: com.example
   - **ArtifactId，也就是name**: vhr
   - **Version**: 1.0.0
3. 点击 Finish，IDEA 会生成一个空的 Maven 项目

### 2.1.2 修改 pom.xml

打开项目根目录下的 `pom.xml`，替换为如下内容。核心是通过 `<parent>` 继承 `spring-boot-starter-parent`，这样所有 Spring Boot 依赖的版本就自动管理了，不用逐个写 `<version>`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <!-- 继承 Spring Boot 父项目：自动管理所有 Spring Boot 依赖的版本 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.4.0</version>
        <relativePath/>
    </parent>

    <groupId>com.example</groupId>
    <artifactId>vhr</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>

    <properties>
        <java.version>8</java.version>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-security</artifactId>
        </dependency>
        <dependency>
            <groupId>org.mybatis.spring.boot</groupId>
            <artifactId>mybatis-spring-boot-starter</artifactId>
            <version>1.3.1</version>
        </dependency>
        <dependency>
            <groupId>com.alibaba</groupId>
            <artifactId>druid</artifactId>
            <version>1.0.29</version>
        </dependency>
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <scope>runtime</scope>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <finalName>vhr</finalName>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
```

这些都是常规的依赖：Spring Boot Web（接口服务）、Spring Security（权限控制）、MyBatis（ORM）、Druid（数据库连接池）、MySQL 驱动。

> **关键说明**：
> - `<parent>` 标签是 Spring Boot 项目的核心——它继承了 `spring-boot-starter-parent`，自动帮你管理所有 Spring Boot 相关依赖的版本，不需要在每个 `<dependency>` 里写 `<version>`
> - 非 Spring Boot 官方 starter（如 `mybatis-spring-boot-starter`、`druid`）需要手动写版本号
> - `spring-boot-maven-plugin` 负责把项目打成可执行的 jar（内置 Tomcat）

### 2.1.3 创建项目目录结构、启动类和配置文件

空的 Maven 项目只有 `src/main/java` 和 `src/main/resources`。你需要手动创建：

1. 在 `src/main/java` 下创建包 `com.example.vhr`
2. 在包里创建启动类 `VhrApplication.java`：

```java
package com.example.vhr;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class VhrApplication {
    public static void main(String[] args) {
        SpringApplication.run(VhrApplication.class, args);
    }
}
```

3. 在 `src/main/resources` 下创建 `application.properties`，配置数据库连接（下面 2.2 节会把 MySQL 跑起来）：

```properties
spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
spring.datasource.url=jdbc:mysql://127.0.0.1:3306/vhr?useUnicode=true&characterEncoding=UTF-8
spring.datasource.username=root
spring.datasource.password=123

server.port=8082
```

## 2.2 用 Docker 启动 MySQL 数据库（数据持久化）

> 💡 我们用 Docker 跑一个 MySQL 8.0，把数据存在你电脑上，**下次关机重启数据还在**。

### 2.2.1 启动 MySQL 容器

在终端执行这一条命令（**只需要执行一次，以后直接 `docker start mysql-vhr` 就能开启：

```bash
docker run -d \
  --name mysql-vhr \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123 \
  -v mysql-vhr-data:/var/lib/mysql \
  mysql:8.0 \
  --default-authentication-plugin=mysql_native_password \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

> - `--default-authentication-plugin=mysql_native_password` 是 8.0 必需的，vhr 项目里的老 JDBC 驱动不认识 8.0 的新密码认证方式

**Windows（cmd / PowerShell）版：**

```powershell
docker run -d ^
  --name mysql-vhr ^
  -p 3306:3306 ^
  -e MYSQL_ROOT_PASSWORD=123 ^
  -v mysql-vhr-data:/var/lib/mysql ^
  mysql:8.0 ^
  --default-authentication-plugin=mysql_native_password ^
  --character-set-server=utf8mb4 ^
  --collation-server=utf8mb4_unicode_ci
```

每行参数解释：

| 参数 | 作用 |
| :--- | :--- |
| `-d` | 后台运行 |
| `--name mysql-vhr` | 给容器起名叫 mysql-vhr，方便以后操作 |
| `-p 3306:3306` | 把容器的 3306 端口映射到你电脑的 3306 端口 |
| `-e MYSQL_ROOT_PASSWORD=123` | root 用户密码设为 123（和 `application.properties` 里一致） |
| `-v mysql-vhr-data:/var/lib/mysql` | **关键**：把 MySQL 数据存到 Docker 命名卷里，容器删了数据都在 |
| `mysql:8.0` | 使用 MySQL 8.0 镜像（有原生 ARM64 和 x64 版本，Mac M 系列和 Windows 都能直接跑） |
| `--default-authentication-plugin=mysql_native_password` | 8.0 需要，让它用老的密码认证方式，老 JDBC 驱动能连上 |
| 最后两行字符集 | 设置数据库默认字符集为 utf8mb4，支持中文 |

### 2.2.2 创建 vhr 数据库

等待 10~30 秒（MySQL 第一次启动比较慢），然后执行：

```bash
docker exec -it mysql-vhr mysql -uroot -p123 -e "CREATE DATABASE IF NOT EXISTS vhr DEFAULT CHARACTER SET utf8mb4;"
```

看到提示 `Query OK, 1 row affected` 就好了。

### 2.2.3 导入 vhr.sql 建表

`vhr` 数据库现在是空的，下一步把表和初始数据导进去。完整的建表脚本在项目里：`resources/vhr.sql`。

**在项目根目录**（包含 `resources/` 文件夹的那一层）打开终端，执行下面的命令。

**Mac / Linux：**

```bash
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources/vhr.sql
```

**Windows（PowerShell）：**

```powershell
Get-Content resources\vhr.sql | docker exec -i mysql-vhr mysql -uroot -p123 vhr
```

**Windows（cmd）：**

```cmd
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources\vhr.sql
```

> 💡 这一步可能需要几十秒（vhr.sql 比较长），终端没有任何报错输出就表示成功了。

### 2.2.4 验证表建好了

执行下面的命令，看看表是不是都建好了：

```bash
docker exec -it mysql-vhr mysql -uroot -p123 -e "USE vhr; SHOW TABLES;"
```

正常会看到 20 张左右的表（hr、role、menu、hr_role、menu_role、department、employee 等等）。

再确认一下权限核心表有数据：

```bash
docker exec -it mysql-vhr mysql -uroot -p123 -e "USE vhr; SELECT COUNT(*) AS hr_count FROM hr; SELECT COUNT(*) AS role_count FROM role;"
```

能看到非 0 的数字就说明表里有初始数据，接下来就可以启动 Spring Boot 项目连数据库了。

### 2.2.5 如果表建错了，想从头来

```bash
# 1. 删掉数据库
docker exec -it mysql-vhr mysql -uroot -p123 -e "DROP DATABASE vhr;"

# 2. 重新建库
docker exec -it mysql-vhr mysql -uroot -p123 -e "CREATE DATABASE IF NOT EXISTS vhr DEFAULT CHARACTER SET utf8mb4;"

# 3. 重新导入 SQL（Mac/Linux 版，Windows 用上面对应平台的命令）
docker exec -i mysql-vhr mysql -uroot -p123 vhr < resources/vhr.sql
```

### 2.2.6 常用命令备忘

| 操作 | 命令 |
| :--- | :--- |
| 查看 MySQL 容器状态 | `docker ps`（能看到 mysql-vhr 就表示在跑） |
| 查看启动日志（排错用） | `docker logs mysql-vhr` |
| 停止 MySQL | `docker stop mysql-vhr` |
| 启动 MySQL（下次开机后） | `docker start mysql-vhr` |
| 重启 MySQL | `docker restart mysql-vhr` |
| 删除容器（数据还在） | `docker rm -f mysql-vhr` |
| 彻底删除数据卷（慎用） | `docker volume rm mysql-vhr-data` |
| 删除 MySQL 镜像（需先删容器） | `docker rmi mysql:8.0` |

## 2.3 创建测试 Controller 并验证

> 💡 Controller 是干什么的？它决定了"用户访问哪个网址 → 程序返回什么内容"。我们先写一个最简单的测试接口，验证项目能不能跑起来。

### 2.3.1 创建 HelloController.java

1. 在 IDEA 左侧找到目录：`src → main → java → com.example.vhr`（这里已经有 `VhrApplication.java`）
2. **在 `vhr` 文件夹上右键** → New → **Java Class**
3. Name 填 `HelloController` → 回车
4. IDEA 会创建一个空文件，把里面的内容全部替换为：

```java
package com.example.vhr;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello Spring Boot!";
    }
}
```

创建完成后，目录结构应该是这样的：

```
src/main/java/com/example/vhr/
├── VhrApplication.java       ← 启动类（程序从这里运行）
└── HelloController.java      ← 测试接口（访问 /hello 返回文字）
```

### 2.3.2 启动项目

确保 MySQL 容器在跑（`docker ps` 能看到 `mysql-vhr`），然后启动 Spring Boot：

- **方式一（推荐）**：打开 `VhrApplication.java`，点击 main 方法左边的绿色 ▶ 按钮 → Run
- **方式二**：在项目根目录的终端执行：`mvn spring-boot:run`

启动成功的标志：控制台有一大段日志，能看到 `Tomcat started on port(s): 8082` 这样的字样，没有大片红色报错。

> 💡 **如果看到"Failed to configure a DataSource"报错**：说明 MySQL 容器没启动，或者 `vhr` 数据库没创建。回到 2.2 节，先 `docker start mysql-vhr`，再确认 `vhr` 库存在。

### 2.3.3 浏览器访问测试

打开浏览器访问：`http://localhost:8082/hello`

- 你会看到一个 **登录页面**（网址自动跳到了 `/login`）→ ✅ **这是正常的**。因为我们在 `pom.xml` 里引入了 Spring Security，它默认会拦截所有请求要求登录。这恰恰证明 Spring Boot + Security + MySQL 都正常启动了。
- 如果你想先**绕开登录直接看到接口返回**，临时在 `pom.xml` 里把 `<dependency>spring-boot-starter-security</dependency>` 这一段注释掉（用 `<!-- -->` 包起来），重启项目再访问 `http://localhost:8082/hello`，就能直接看到 `Hello Spring Boot!` 这几个字。确认环境没问题后记得改回来。

OK，至此，我们的工程就创建并验证完成。
